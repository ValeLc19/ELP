ELP · Deployment in progress

Putting three cloud services behind your events app

ELP started as a website with no backend — saved events and followed businesses lived only in your browser. We're wiring it to a real backend so that data lives in a database and follows you across devices. That takes three managed services, each doing one job.

02

Why it's split this way

Cloudflare holds the site

A React app is just files. A static host serves them cheaply and fast — no server needed for the page itself.

Render holds the logic

A browser can't be trusted with everyone's data. The API is the gatekeeper: it verifies who you are, then reads/writes the database on your behalf.

Supabase holds the truth

One place for the real data and for login. It signs a token when you sign in; the backend checks that signature against Supabase's public key.

The one thing blocking us
