WHAT RUNS ON ITS OWN
sync-events
cron: 0 11 * * *
Re-scrapes visitelpaso daily and upserts into Postgres. Owns and prunes only its own rows, so the curated seed survives as a fallback.
keepalive
cron: 0 12 1,7,13,19,25 * *
Pings /health every ~6 days. Free-tier Supabase auto-pauses after 7 idle days; this tiny DB touch counts as activity.
CONFIG THAT BIT US — KEEP IT THIS WAY
Render needs SUPABASE_URL
Logins are signed ES256 and verified against that project's JWKS. Without it, every authenticated request 401s. It is not a secret — it is the public project URL.
Use the apex URL, not the preview
Every npm run deploy also publishes a <hash>.elpaso-events.pages.dev preview — which is what Cloudflare shows you afterwards. Easy to test the wrong site. CORS covers both via regex.
"Works local, broken prod" is misleading
Most user data is local-first, so server 401s fail silently. Only /pipeline/extract hard-depends on the backend — debug prod auth there, not on saves.
STILL OPEN
DEFERRED Auto-scrape social posts
Paste-a-link extraction ships today. Full Instagram / Facebook / TikTok auto-pull needs a vision LLM (paid key) and a way past each platform's login wall.
LIMIT Email deliverability
Supabase's free-tier sender is rate-limited and can land in spam. Fine for a demo; a real launch wants custom SMTP.
